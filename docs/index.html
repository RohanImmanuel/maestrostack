Run Maestro tests on real devices with one command

MaestroStack packages your flow folders, uploads the app and test suite, and starts a BrowserStack build across every device you list. From a single config file. No glue scripts.

Get started See how it works
npm install -g maestrostack click to copy
maestrostack 

          

        

      


    


    
    

      
      

        

          The difference
          
Eight manual steps to just one

          

            By hand, BrowserStack's Maestro flow is eight separate steps, every single
            time. MaestroStack folds all of them into a single command.
          

        


        

          

          

            
            maestrostack run
            one command
          


          

            1
            Upload app
          

          

            2
            Zip suite
          

          

            3
            Upload zip
          

          

            4
            Copy app URL
          

          

            5
            Copy suite URL
          

          

            6
            Build payload
          

          

            7
            Set devices
          

          

            8
            Call endpoint
          


          

        

      

    


    
    

      
      

        

          How it works
          
From a folder of flows to a live build

          
Each run walks the same path. You describe the what, MaestroStack handles the how.

        


        
    
          
  1. 
            
    
              
            
    
            
    
              01
              
    Describe it in one file
    
              
    A single maestrostack.yml holds your platform, app, devices and which flows to run.
    
            
    
          
    2. 

          
  2. 
            
    
              
            
    
            
    
              02
              
    Flows get packaged
    
              
    MaestroStack finds your flow files in subfolders and zips them with the structure BrowserStack expects.
    
            
    
          
    3. 

          
  3. 
            
    
              
            
    
            
    
              03
              
    App and suite upload
    
              
    Your binary and the suite zip are sent to BrowserStack, or it reuses an app you already uploaded.
    
            
    
          
    4. 

          
  4. 
            
    
              
            
    
            
    
              04
              
    Builds run on real devices
    
              
    One build request fans out across every device in your list, Android or iOS.
    
            
    
          
    5. 

          
  5. 
            
    
              
            
    
            
    
              05
              
    You get a build ID
    
              
    MaestroStack prints a clean summary and the BrowserStack build ID to follow your run.
    
            
    
          
    6. 
        

      

    


    
    

      
      

        

          What you get
          
Built to be safe, clear and repeatable

        


        

          

            
            
Config driven

            
One YAML file is the source of truth. Swap configs per platform or environment with a flag.

          


          

            
            
Dry run first

            
Preview the exact files and BrowserStack payload before a single byte leaves your machine.

          


          

            
            
Many devices

            
List as many real devices as you need. One run, every device, Android and iOS.

          


          

            
            
Works everywhere

            
The same behaviour on macOS, Linux and Windows. No platform surprises in CI.

          


          

            
            
Secrets stay secret

            
Credentials load from env or a .env file, are validated up front, and never printed.

          


          

            
            
Clear errors

            
Missing device, bad path or absent flow fails early with a message you can act on.

          

        

      

    


    
    

      
      

        

          Safe by default
          
Look before you launch

          

            maestrostack run --dry-run validates everything, shows the flows it
            would package and prints the build payload with secrets hidden. Nothing is uploaded.
          

          Read the run docs
        


        

          

            

              
              maestrostack run --dry-run
            


Dry run only. No API calls made.

Would package:
- smoke/login.yml
- regression/checkout.yml

Would call:
POST /app-automate/maestro/v2/android/build

Payload:
{
  "app": "<resolved after upload>",
  "testSuite": "<resolved after upload>",
  "project": "Maestro_Test",
  "devices": ["Google Pixel 7-13.0"],
  "execute": ["smoke/login.yml"]
}
